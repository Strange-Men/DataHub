# P1 Workflow UX Test Checklist — 第一次使用者 UX 测试清单

> 版本：P1-M20.5  
> 目标：验证 4 主流程 UX 对首次使用者的可用性

## 1. 入口与导航

- [ ] 用户进入 P1 页面后，是否一眼看到 4 个主流程步骤指示器？
- [ ] 当前是 4 个步骤而不是 7 个步骤？
- [ ] 每个步骤是否有清晰的标题和简短说明？

## 2. Step 1 — 导入数据

- [ ] 用户是否知道从哪里开始？（上传 JSON / 拖拽 / 示例数据 / 粘贴 JSON）
- [ ] 拖拽 JSON 文件是否生效？
- [ ] 点击「使用示例数据」是否自动填充？
- [ ] 「高级粘贴 JSON」是否可以展开/收起？
- [ ] 导入成功后是否显示 batch_id 和消息数量？
- [ ] 导入成功后是否有「下一步：去清洗数据」按钮？
- [ ] 导入失败是否有明确的错误提示？

## 3. Step 2 — 清洗数据

### 3A. 机器清洗

- [ ] 是否清楚显示「机器清洗」标签页？
- [ ] 点击「执行机器清洗」是否有响应？
- [ ] 清洗完成后是否显示清洗摘要指标？
- [ ] 是否不再有「查看清洗结果」这种点了没反馈的按钮？
- [ ] 清洗完成后是否有「进入人工清洗工作台」按钮？

### 3B. 人工清洗工作台

- [ ] 消息列表是否有分页（默认每页 10 条）？
- [ ] 分页是否包含上一页/下一页和页码？
- [ ] 是否有搜索框可以搜索消息内容？
- [ ] 是否有质量等级筛选（全部/高质量/需复核/低质量）？
- [ ] 是否有处理方式筛选（保留/修改后保留/丢弃/需复核）？
- [ ] 每条消息默认是否只显示摘要（角色、质量、前80字）？
- [ ] 点击摘要行是否可以展开/收起详情？
- [ ] 处理进度条是否显示（已处理/总数）？
- [ ] 保存后是否有明显的反馈信息（已保存、处理方式、下一步）？
- [ ] 是否有「下一步：生成并审核知识」按钮？

## 4. Step 3 — 生成并审核知识

### 4A. 生成待审核知识

- [ ] 是否显示「生成待审核知识」（不是「知识抽取」）？
- [ ] 点击「生成待审核知识」是否有响应？
- [ ] 生成后是否显示生成数量和状态？
- [ ] 生成后是否有「进入知识审核」按钮？
- [ ] 如果未完成清洗是否给出提示？

### 4B. 知识审核

- [ ] 是否有审核状态筛选（待审核/已通过/已驳回/打回修改）？
- [ ] 是否有质量等级筛选？
- [ ] 是否有意图筛选？
- [ ] 是否有关键词搜索？
- [ ] 是否有状态统计栏（待审核/已通过/已驳回/打回修改/总计）？
- [ ] 知识列表是否有分页（每页 10 条）？
- [ ] 点击「审核通过」是否有反馈？
- [ ] 审核通过后是否提示「该知识可同步到 RAG 知识库」？
- [ ] 驳回和打回修改是否有反馈？
- [ ] 保存修改是否有反馈？
- [ ] 没有已通过知识时是否阻止进入 Step 4？

## 5. Step 4 — 更新知识库并测试 Agent

### 5A. 同步已审核知识

- [ ] 文案是否是「同步已审核知识到 RAG 知识库」（不是「构建知识库/Build RAG」）？
- [ ] 是否有说明这是增量更新而非新建知识库？
- [ ] 同步后是否显示已同步知识块数量？
- [ ] 如果没有已通过知识，按钮是否禁用并有提示？

### 5B. 测试 Agent 回答

- [ ] 是否只有一个主按钮「测试 Agent 回答」？
- [ ] 输入问题后点击按钮是否有响应？
- [ ] 搜索后是否显示命中知识数量？
- [ ] 搜索后是否显示 Agent 回答？
- [ ] 搜索后是否显示引用来源列表（含相关度和知识块内容）？
- [ ] 搜索后是否显示 retrieval_id？
- [ ] 如果未命中知识是否显示提示：「请先确认已有审核通过的知识，并已同步到 RAG 知识库」？

### 5C. 提交 Bad Case

- [ ] 是否有 retrieval_id 后才显示 Bad Case 提交区域？
- [ ] 提交后是否提示「已提交 Bad Case，并生成待审核知识草稿」？
- [ ] 提交后是否提示「可回到 Step 3 审核」？
- [ ] 是否有回到 Step 3 的快捷按钮？

## 6. 全局 UX

- [ ] 每个主流程是否有当前状态标识？
- [ ] 每个按钮点击后是否都有反馈（成功/失败）？
- [ ] 页面是否没有暴露 DATABASE_URL / API Base URL / 数据库细节等技术信息？
- [ ] 是否没有一堆卡片无限堆叠（有分页控制）？
- [ ] 用户是否不需要回跳找按钮（每步都有下一步指引）？
- [ ] 刷新页面后数据是否仍然存在（数据库持久化）？

## 7. 禁止项检查

- [ ] 没有恢复「高级信息」页面
- [ ] 没有把每个后端动作做成独立的大 Step
- [ ] 没有 7 个主流程（必须是 4 个）
- [ ] Step 2 中不藏「知识抽取」
- [ ] 「知识抽取」已统一改为「生成待审核知识」
- [ ] 「构建知识库 / Build RAG」已统一改为「同步已审核知识到 RAG 知识库」
