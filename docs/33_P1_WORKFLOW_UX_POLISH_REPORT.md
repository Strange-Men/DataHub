# P1 Workflow UX Polish Report

> 版本：P1-M20.5  
> Commit: `[P1-M20.5] polish: simplify P1 workflow user experience`  
> 日期：2026-07-05

## 概述

将 P1 客服文本中台从 **5 个独立 Step 的流程屎山** 重构为 **4 个主流程 + 内部子操作** 的简洁工具流程。

## 修改清单

### 1. 步骤结构重组

| 旧结构 | 新结构 |
|---|---|
| Step 1: 导入数据 | Step 1: 导入数据 |
| Step 2: 机器清洗 | Step 2: 清洗数据（A.机器清洗 B.人工清洗工作台） |
| Step 3: 人工清洗 | Step 3: 生成并审核知识（A.生成待审核知识 B.知识审核） |
| Step 4: 知识审核 | Step 4: 更新知识库并测试 Agent（A.同步 B.测试 C.Bad Case） |
| Step 5: RAG & Agent | （合并入 Step 4） |

### 2. 文案统一

- "知识抽取" → "生成待审核知识"
- "构建知识库 / Build RAG" → "同步已审核知识到 RAG 知识库"
- 明确说明"不是新建知识库，而是更新同一个 RAG 知识库"

### 3. 人工清洗工作台增强

- 新增分页（默认每页 10 条）
- 新增搜索框（按消息内容搜索）
- 新增质量等级筛选（全部/高质量/需复核/低质量）
- 新增处理方式筛选（保留/修改后保留/丢弃/需复核）
- 每条消息默认只显示摘要，点击展开详情
- 新增处理进度条
- 保存后有明确反馈：已保存、处理方式、下一步提示
- 删除"查看清洗结果"这种无反馈按钮，机器清洗后自动加载清洗数据

### 4. 知识审核增强

- 新增审核状态统计栏（待审核/已通过/已驳回/打回修改/总计）
- 新增分页（默认每页 10 条）
- 审核通过后提示"该知识可同步到 RAG 知识库"
- 无已通过知识时阻止进入 Step 4

### 5. Agent 测试增强

- 测试区只保留一个主按钮"测试 Agent 回答"
- 新增结果面板：
  - 命中知识数量
  - Agent 回答
  - 引用来源列表
  - retrieval_id
- 未命中时给出明确提示
- Bad Case 提交后提示可回到 Step 3 审核

### 6. 全局 UX 改进

- 每个主流程有子标签页切换
- 每个操作有成功/失败反馈
- 每一步有明确的下一步指引按钮
- 不会有按钮点了没反应的情况
- 不会无限堆叠卡片
